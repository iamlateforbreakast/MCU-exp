# Issues boot Fedora 43 live USB image

Potential Causes and Solutions
USB Boot is Disabled in BIOS: Many Dell Latitude models ship with external USB ports disabled for booting to prevent unauthorised OS installations.
Fix: Restart and tap F2 to enter BIOS. Navigate to System Configuration > USB Configuration and ensure Enable External USB Port and Enable USB Boot Support are checked.

Secure Boot Interference: While Fedora 43 supports Secure Boot, sometimes a specific hardware/firmware combination on a second-hand machine might block "untrusted" media.
Fix: In the BIOS, go to Secure Boot and set it to Disabled temporarily to see if the drive appears.

Fastboot Setting: If "Fastboot" is set to "Minimal" in the BIOS, it may skip initializing USB ports during the boot process to save time.
Fix: In the BIOS, go to POST Behavior > Fastboot and select Thorough.

Storage Controller Mode (RAID vs. AHCI): Dells often ship in RAID mode, which can cause issues with Linux installers recognizing the drive even if the USB boots.
Fix: In the BIOS, under Storage, change the SATA/NVMe Operation from RAID to AHCI/NVMe.

USB Formatting (UEFI/GPT): If the USB works on an older computer but not this one, it might be formatted with an MBR partition table for Legacy BIOS. The Latitude 7340 requires a GPT partition table for UEFI.
Fix: Re-create your live media using the Fedora Media Writer or Rufus (selecting GPT and UEFI target system).

# Configure Toolbx on Fedora for Hardcaml development

you need to create a container environment, install the OCaml package manager (opam), and then set up the Hardcaml-specific libraries and FPGA tools.

## 1. Create and Enter the Toolbx Container

First, create a dedicated container to keep your Hardcaml tools separate from your host system.

* Create the container: Run toolbox create -c hardcaml-dev.
* Enter the container: Use toolbox enter -c hardcaml-dev. Your prompt will change (usually prefixed with ⬢) to show you are inside. [2, 3, 4] 

## 2. Install OCaml and Opam

Inside the Toolbx environment, you must install the base development tools and the OCaml package manager.

* Install dependencies: Run sudo dnf install opam make gcc m4.
* Initialise opam: Run opam init. When prompted to update your shell profile, choose "yes" so that the environment variables are set correctly for future sessions.
* Update environment: Run eval $(opam env) to apply the changes to your current session. [3, 6, 7] 

## 3. Install Hardcaml and Tooling

Once opam is ready, you can install the Hardcaml library and its associated tools for hardware simulation and generation.

* Install Hardcaml: Run opam install hardcaml.
* Install Extras: For testing and waveform viewing, you typically need:
* opam install hardcaml_waveterm (for terminal-based waveforms).
   * opam install ppx_hardcaml (for Hardcaml-specific syntax extensions).

## 4. Optional: FPGA Simulation and Synthesis Tools

If you need to simulate Verilog/VHDL generated by Hardcaml or perform synthesis:

* Verilator/Icarus: Install them via sudo dnf install verilator iverilog inside the toolbox.
* GTKWave: If you need a GUI waveform viewer, install it with sudo dnf install gtkwave. Toolbx allows GUI applications to run and display on your host desktop. [4, 8] 

## 5. IDE Integration (VS Code)

To use VS Code with your Toolbx-based Hardcaml setup:

   1. Host Side: Install the OCaml Platform extension in VS Code.
   2. Toolbx Side: Ensure the OCaml Language Server is installed inside the container: opam install ocaml-lsp-server.
   3. Connection: You can configure VS Code to use the toolbox environment by prefixing the OCaml LSP path with toolbox run -c hardcaml-dev or by using the Dev Containers extension with Podman. [6, 9, 10, 11, 12] 

Pro-tip: Since Toolbx shares your home directory, your .opam folder and project files will persist even if you delete and recreate the container. [3, 4, 13, 14]
